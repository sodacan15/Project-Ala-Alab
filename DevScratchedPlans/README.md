# Documentation Directory

This directory contains project documentation and supporting notes that are kept for development, architecture, and memory purposes.

- `protocol/` — design notes, agent instructions, architecture diagrams, and protocol files used to define how the local TripleTalk relay and human-mediated agent flow operate.
- `dev-notes/` — developer artifacts, methodology documents, and context notes created during the project build that are useful for future maintenance or handoff.
- `assets/` — static assets, diagrams, or visual resources referenced by docs or the project narrative.
- `agent-memory/` — memory artifacts for agent workflows, preserved separately to keep the documentation directory organized and clearly scoped.
